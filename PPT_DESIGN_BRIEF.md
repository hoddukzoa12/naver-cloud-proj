### 02 문제와 사용자

- **자료**:

- **디자인 힌트**: 분산된 사이트 아이콘(한국장학재단·대학교·재단) → 중앙 화살표 수렴 구조

  [사용자 조건 입력]

  [HyperCLOVA X + DB 1,850건 검색]

  - 챗봇 맞춤 추천
  - 키워드·학자금 구간·상태 필터
  - CLOVA Studio 공고문 요약
  - 공식 사이트 직접 링크

### 04 Ncloud 아키텍처
- **핵심 메시지**: "Ncloud 4개 서비스로 전체 파이프라인 구성"
- **자료**: 서비스 연결 다이어그램 (텍스트 ASCII로 표현)
  ```
  [사용자 브라우저]
       |
  [Ncloud VPC Server — Node.js 18, Port 80]
       |─── POST /api/chat ──→ [HyperCLOVA X HCX-DASH-001]
       |─── POST /api/summary ─→ [CLOVA Studio Summarization v2]
       └─── DB 쿼리 ──────────→ [PostgreSQL — 장학금 1,850건]

  [공공데이터 API (odcloud.kr)]
       └── scraper.js → PostgreSQL (수집 파이프라인)
  ```
- 서비스별 선택 이유 표:

  | 서비스 | 사용 이유 |
  |--------|-----------|
  | HyperCLOVA X | 한국어 특화 LLM — 장학금 한국어 질의응답 품질 |
  | CLOVA Studio | 같은 API 키로 요약·채팅 통합, Ncloud 일원화 |
  | VPC Server | Node.js 앱 호스팅, 포트 80 외부 개방 |
  | PostgreSQL | 1,850건 공공데이터 영구 저장·실시간 조회 |

- **주의**: 서버 IP, DB 접속 정보 절대 표시 금지

### 05 AI 기능 1 — HyperCLOVA X 챗봇
- **핵심 메시지**: "DB 컨텍스트 + LLM — 사실 기반 추천만 한다"
- **자료**:
  - 입력: 사용자 메시지 + DB 장학금 컨텍스트 (키워드 매칭 상위 30건)
  - 처리: system 프롬프트 (공식 목록 외 임의 생성 금지 지시) + user 메시지
  - 출력: AI 추천 텍스트 + 관련 장학금 카드 (최대 3개)
  - 파라미터: maxTokens 1024 · temperature 0.5 · timeout 30초
  - 실패 처리:

    | 상황 | 동작 |
    |------|------|
    | API 키 없음 | DB 조회 결과 상위 3건 목록 텍스트로 fallback |
    | 타임아웃(30초) | fallback 전환, 서비스 중단 없음 |
    | HTTP 오류 | fallback 전환, warning 필드 반환 |

- **디자인 힌트**: 입력→처리→출력 3칸 레이아웃. 실제 채팅 말풍선 UI 스크린샷 1장 포함 권장

---

### 06 AI 기능 2 — CLOVA Studio 요약
- **핵심 메시지**: "긴 공고문을 3문장으로 — API 없어도 데모 동작"
- **자료**:
  - 엔드포인트: `POST /v1/api-tools/summarization/v2`
  - 입력: 장학금명 + 기관 + 자격조건 텍스트 (최대 35,000자)
  - 출력: 요약문 (`result.text`) + 입력 토큰 수
  - 요청 형태: `autoSentenceSplitter: true`, `segMaxSize: 1000`
  - fallback: API 키 없으면 앞 3문장 추출 반환 (데모 항상 동작)
- **디자인 힌트**: SummarySheet 로딩→완료 2단계 화면 스크린샷 나란히 배치

### 11 오류 수정 기록
- **핵심 메시지**: "발견→분석→수정 — 실제 디버깅 과정"
- **자료**: 5건 수정 이력 (원인·수정 방법 포함)

  | 버그 | 원인 | 수정 |
  |------|------|------|
  | 모든 장학금 "미검증" 표시 | SELECT에 source_url 누락 → mapDbRow가 항상 undefined | 3개 쿼리 모두 source_url 컬럼 추가 |
  | 공식 링크 클릭 시 toast | noticeAction이 window.open 대신 showToast 호출 | window.open(_blank, noreferrer) 으로 변경 |
  | 외부 접속 불가 | server.listen('127.0.0.1') — localhost만 바인딩 | '0.0.0.0' 으로 변경 |
  | 시드 데이터 목록에 노출 | 가짜 기관 s1~s7이 DB에 포함 | 전체 쿼리에 AND source_url IS NOT NULL 추가 |
  | 챗봇이 최근 10건만 조회 | LIMIT 10, 단순 정렬 | 키워드 OR 검색 + LIMIT 30, fallback LIMIT 20 |

### 12 보안 관리
- **핵심 메시지**: "API 키·서버 접속 정보는 코드에 없다"
- **자료**:
  - 비밀값 관리 원칙: `.env` 파일만 사용, 코드·PPT·화면 공유 시 노출 없음
  - 프론트엔드는 `/api/summary`, `/api/chat` 만 호출 (API 키 직접 접촉 없음)
  - 서버만 환경변수로 API 키 읽음
  - 보안 구조 요약:
    ```
    브라우저 ──→ /api/chat, /api/summary (키 없음)
    server.js ──→ process.env.CLOVA_STUDIO_API_KEY (서버 환경변수)
    코드·PPT ──→ 키값 없음
    ```
- **절대 표시 금지**: API 키 값, 서버 접속 비밀번호, DB 비밀번호, SSH 개인키
- **디자인 힌트**: 열쇠·자물쇠 아이콘 활용. 마케팅 문구 없이 구조도로만 표현.
