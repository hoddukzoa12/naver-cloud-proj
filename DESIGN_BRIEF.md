# DESIGN_BRIEF.md — Scholarship Radar (Claude Design 붙여넣기용)

> 디자인 톤: 공공 서비스 느낌 — 신뢰감 있고 단정하게. 차분한 블루 계열, 한국어 UI 전용, 카드 기반 레이아웃, 모바일 반응형.

## 1. 프로젝트 한 줄 설명

HyperCLOVA X 기반 대학생 장학금 실시간 맞춤 검색 서비스 — 사용자 조건을 입력하면 공식 공고 우선 원칙으로 맞는 장학금을 찾아준다.

## 2. 주요 사용자

- 국내 대학교 학부생·대학원생
- 모바일 비중 높음
- 처음 쓰는 사람도 바로 이해할 수 있어야 함

## 3. 핵심 화면 목록

1. 메인 검색 화면
2. 장학금 상세 패널
3. 요약 결과 화면
4. 미검증 경고 배너 (카드 상단 공통 컴포넌트)

## 4. 각 화면의 목적

| 화면 | 목적 |
|------|------|
| 메인 검색 | 조건 입력 후 모집 중인 장학금을 카드 리스트로 한눈에 보여준다. 첫 화면에서 검색창, 모집 중 장학금 수, 필터 버튼(학교·구간·유형), 상위 카드 2~3개가 바로 보여야 한다 |
| 장학금 상세 패널 | 자격·금액·마감·링크 카드를 보여주고 CLOVA 요약 버튼으로 공고문 요약을 요청한다 |
| 요약 결과 | CLOVA Studio 요약 텍스트·출처·fallback 경고를 표시한다 |
| 미검증 경고 배너 | `verification_status !== 'verified'` 항목임을 카드 상단에서 알려 사용자가 공식 공고를 직접 확인하도록 유도한다 |

## 5. 입력값과 결과값

**입력 필드**
- 키워드
- 학교명
- 학자금 지원구간 (1~10)
- 모집 상태 (전체 / 모집중 / 예정 / 마감)
- 장학금 유형 (등록금 / 생활비 / 혼합)

**결과 화면에 표시할 정보**
- 장학금명
- 운영기관
- 금액 (학기당·연간)
- 마감일
- 모집 상태 배지
- 미검증 경고 (해당 시)
- 공식 공고 링크 (공고 URL과 신청 URL은 분리 제공)

## 6. 3분 발표 최소 흐름: 입력 → 처리 중 → 결과 → 다시 시도

| 단계 | 화면에 보여야 할 것 |
|------|---------------------|
| ① 입력 | 검색창 + 필터(학교·구간·유형·모집 상태) + 모집 중인 장학금 수 |
| ② 처리 중 | 카드 스켈레톤 UI (로딩 상태) |
| ③ 결과 | 결과 카드 리스트 → 카드 선택 시 상세 패널 → "CLOVA 요약" 버튼 → 요약 결과 + 출처 표시 |
| ④ 다시 시도 | 빈 상태/오류 시 안내 문구와 함께 필터 조정 후 재검색이 한 동작으로 가능해야 함 |

## 7. 로딩 / 오류 / 빈 상태

- **로딩**: 카드 스켈레톤 UI
- **오류**: "요약 실패, 공식 공고를 직접 확인하세요"
- **빈 상태**: "조건에 맞는 장학금이 없습니다 — 필터를 조정해보세요"

## 8. 화면 밀도

**권장: 실제 앱처럼 보이되 화면 수는 발표용으로 최소화** — 3분 발표에서 신뢰감(공공 서비스 톤)을 주려면 실제 서비스 수준의 카드 밀도가 필요하지만, 흐름은 위 4단계만 매끄럽게 이어지면 충분하다.

## 9. 버튼 우선순위

1. **검색 (가장 먼저 눌러야 할 버튼)** — 첫 화면 최상단 검색창과 함께
2. 공식 공고 보기
3. CLOVA 요약
4. 신청하기 (상세 패널 하단 고정)

## 10. 모바일에서 가장 중요한 행동

- **"공식 공고 보기"** — 항상 엄지가 닿는 위치에 배치
- **"신청하기"** — 상세 패널 하단 고정

## 11. 더미 데이터 규칙

- 장학금 샘플 데이터는 **기관·공고 단위 정보만** 사용한다: 장학금명, 운영기관, 금액, 마감일, 모집 상태
- 실제 데이터처럼 보이는 더미 개인정보(이름, 학번 등)는 절대 넣지 않는다
- 미검증(`verified` 아님) 샘플에는 경고 배너가 보이도록 최소 1건 포함한다
- 피해야 할 디자인: 과한 장식(그림자·그라데이션 중첩), 읽기 어려운 저대비 텍스트

## 12. 주의 문구

> ⚠️ **디자인 산출물과 더미 데이터 어디에도 실제 개인정보(이름·주민등록번호·전화번호·이메일), API 키·Bearer 토큰·Secret Key, SSH 키, 금융 데이터(계좌·카드번호), DB 비밀번호를 절대 넣지 말 것.** 모든 비밀값은 서버 환경변수로만 관리한다.
