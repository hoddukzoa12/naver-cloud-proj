# CLAUDE.md — Scholarship Radar

## 1. 프로젝트 한 줄 설명

HyperCLOVA X 기반 대학생 장학금 실시간 맞춤 검색 서비스 — 사용자 조건을 입력하면 공식 공고 우선 원칙으로 맞는 장학금을 찾아준다.

## 2. 대상 사용자와 핵심 기능

**대상**: 국내 대학교 학부생·대학원생

| 기능 | 설명 |
|------|------|
| 장학금 검색·필터 | 키워드·학교·학자금 지원구간·모집 상태·금액·유형으로 필터링 |
| 장학금 상세 조회 | 자격·금액·마감일·기관 정보 카드 표시 |
| 공식 사이트 이동 | 공식 공고 URL과 신청 URL 분리 제공 |
| 공고문 요약 | CLOVA Studio로 긴 모집요강 자동 요약 (키 없으면 로컬 fallback) |

## 3. 기술 스택과 실행 명령

**스택**: 정적 HTML/CSS/JS (프레임워크 없음) · Node.js 18+ 내장 모듈만 · 외부 npm 패키지 금지

```bash
# 개발 서버
node server.js

# 구문 검사
node --check scholarship-core.js && node --check runbook.js && node --check server.js

# 단위 테스트
node --test tests/*.test.mjs

# Smoke test
curl -I http://127.0.0.1:8787/
curl -X POST http://127.0.0.1:8787/api/summary \
  -H 'Content-Type: application/json' -d '{"text":"장학금 공고문"}'
```

## 4. Ncloud AI/인프라 서비스와 역할

| 서비스 | 역할 | 상태 | 환경변수 |
|--------|------|------|----------|
| **HyperCLOVA X** (CLOVA Studio Chat) | 사용자 조건 기반 장학금 맞춤 추천 — 핵심 AI | 미구현 | `CLOVA_STUDIO_API_KEY` |
| **CLOVA Studio** (Summarization) | 장학금 공고문 자동 요약 (`/api/summary` 프록시) | 구현됨 | `CLOVA_STUDIO_API_KEY`, `CLOVA_STUDIO_REQUEST_ID` |
| **Object Storage** | 장학금 JSON 데이터 저장·배포 | 미구현 | — |
| **Server (VPC)** | Node.js 18+ 앱 호스팅 | 미구현 | `PORT` (기본 8787) |
| **DB** | 장학금 메타데이터 영구 저장 | 미구현 | — |

모든 Ncloud API 키는 `server.js` 서버 환경변수에서만 읽는다. API 키가 없으면 각 기능은 로컬 fallback으로 전환되어 데모가 계속 동작한다.

## 5. Claude Designer 참고

### 화면 목록

| 화면 | 주요 컴포넌트 |
|------|--------------|
| 메인 검색 | 필터 폼 + 결과 카드 리스트 |
| 장학금 상세 패널 | 자격·금액·마감·링크 카드 + CLOVA 요약 버튼 |
| 요약 결과 | 요약 텍스트·출처·fallback 경고 |
| 미검증 경고 배너 | `verification_status !== 'verified'` 항목 카드 상단에 표시 |

### 디자인 명세

| 항목 | 내용 |
|------|------|
| 주요 사용자 | 국내 대학교 학부생·대학원생. 모바일 비중 높음, 처음 쓰는 사람도 바로 이해해야 함 |
| 첫 화면에서 바로 보여야 하는 것 | 검색창, 모집 중인 장학금 수, 필터 버튼(학교·구간·유형), 상위 카드 2~3개 |
| 입력 화면에 필요한 필드 | 키워드, 학교명, 학자금 지원구간(1~10), 모집 상태(전체/모집중/예정/마감), 장학금 유형(등록금/생활비/혼합) |
| 결과 화면에 필요한 정보 | 장학금명, 운영기관, 금액(학기당·연간), 마감일, 모집 상태 배지, 미검증 경고, 공식 공고 링크 |
| 로딩/오류/빈 상태 | 로딩: 카드 스켈레톤 UI / 오류: "요약 실패, 공식 공고를 직접 확인하세요" / 빈 상태: "조건에 맞는 장학금이 없습니다 — 필터를 조정해보세요" |
| 모바일에서 가장 중요한 버튼 | "공식 공고 보기" (항상 엄지 닿는 위치), "신청하기" (상세 패널 하단 고정) |
| 피해야 할 디자인 | 과한 장식(그림자·그라데이션 중첩), 읽기 어려운 저대비 텍스트, 실제 데이터처럼 보이는 더미 개인정보(이름·학번 등) |

**디자인 톤**: 공공 서비스 느낌 — 신뢰감 있고 단정하게. 차분한 블루 계열, 한국어 UI 전용, 카드 기반 레이아웃, 모바일 반응형.

## 6. Sub Agent 호출 규칙

세부 역할은 `SUBAGENTS.md` 참조.

| 상황 | Agent |
|------|-------|
| JS/HTML 구현·수정 | `executor` |
| 장학금 데이터·Ncloud API 조사 | `researcher` / `document-specialist` |
| 구현 완료 후 동작 검증 | `verifier` |
| 코드 품질·보안 리뷰 | `code-reviewer` |
| 테스트 작성 | `test-engineer` |
| UI/UX 화면 설계 | `designer` |

구현 Agent와 검증 Agent는 같은 컨텍스트에서 겸하지 않는다 (자기 검증 금지). Ncloud SDK 사용 전 `document-specialist`로 공식 문서 먼저 확인.

## 7. SSH 서버 반영 규칙

- SSH 또는 배포 관련 요청은 반드시 계획부터 시작한다. 바로 명령을 실행하지 않는다.
- 서버 주소·계정명·키 경로·비밀번호·API 키는 CLAUDE.md에 직접 쓰지 않는다.
- 서버 관련 정보는 `SERVER_RUNBOOK.md`에 별도 정리한다 (실행 위치·포트·환경변수·실행 명령·롤백 방법).
- 실제 반영 전 체크리스트:
  1. 로컬에서 `node --check *.js && node --test tests/*.test.mjs` 통과 확인
  2. 비밀값 노출 검사 (`git diff`로 API 키·토큰 포함 여부 확인)
  3. 발표 데모 흐름 동작 확인
- 서버에서 명령을 실행하기 전에 어떤 명령을 왜 실행하는지 사용자에게 먼저 설명한다.

## 8. 금지사항

어떤 파일에도, 어떤 형태로도 하드코딩 금지:

- API 키 / Bearer 토큰 / Secret Key (CLOVA, Ncloud 포함)
- SSH 개인키 / 패스프레이즈
- 실제 개인정보 (이름, 주민등록번호, 전화번호, 이메일)
- 실제 금융 데이터 (계좌번호, 카드번호)
- 데이터베이스 비밀번호 / 연결 문자열

비밀값은 `.env` 또는 서버 환경변수를 사용한다. `.env`는 반드시 `.gitignore`에 포함.

## 9. 요청 말투별 작업 규칙

| 말투 | 처리 방식 |
|------|-----------|
| "가볍게 봐줘", "오탈자만 봐줘" | 메인 Claude가 혼자 확인 |
| "반영해줘", "한 화면 수정해줘" | 파일 먼저 읽고 → 작은 수정 계획 → 진행 |
| "회의해줘", "토론해줘", "깊게 검토해줘" | 필요한 역할만 불러 의견 수렴 (2~5개) |
| "서버", "배포", "보안", "비밀값" 포함 | 바로 수정 금지 → 점검표와 계획 먼저 작성 |

- 외부 확인이 필요한 내용은 /mcp와 /skills를 먼저 확인한다.
- 확인 도구가 없으면 추정하지 말고 "확인이 필요함"을 명시한다.
- 모든 역할을 자동으로 부르지 않는다. 요청에 맞는 역할만 선택한다.
