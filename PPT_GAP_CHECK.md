## 프로젝트 매니저

**빠진 것**

  - 예: 국내 장학금 공고 총 건수, 학생 1인당 평균 탐색 시간 등

**보완 방법**

- 발표 목표를 "MVP 동작 확인"으로 명시하고 다음 단계(배포 후 사용자 피드백 수집) 1문장 추가

## AI 기능 엔지니어

**빠진 것**
1. RAG 컨텍스트 선택 기준이 "LIMIT 30"만 적혀 있고, 왜 이 30건인지(정렬·필터 기준) 설명 없음
2. 실제 입출력 샘플이 없음 — 발표자가 AI가 실제로 뭘 하는지 보여줄 근거 없음
3. HyperCLOVA X 응답 지연(latency) 측정값 없음
4. fallback 전환 조건이 "API 키 없음"만 기재 — 타임아웃(30초), HTTP 비정상 응답도 포함되지만 누락

**보완 방법**
- 서버에서 실제 채팅 1회 실행 후 request/response JSON을 캡처해서 슬라이드 1장 분량으로 첨부
- `server.js:196-253` 의 timeout/error 분기를 표로 요약 (정상·타임아웃·API 오류·키 없음 4가지 경로)
- `searchScholarships` 정렬 기준(키워드 매칭 → deadline ASC → 모집중 우선)을 한 줄로 추가

## Ncloud 설계자

**빠진 것**
1. 서비스 간 연결 구조 다이어그램 없음 (클라이언트 → Node.js → HyperCLOVA X / CLOVA Studio / PostgreSQL 흐름)
2. Object Storage가 CLAUDE.md에 "미구현"으로 있는데, PPT_EVIDENCE.md에서 완전히 빠짐 — 발표자가 질문받을 수 있음
3. DB 스키마(테이블 컬럼 목록)가 없어서 "어떤 데이터를 저장하는가" 설명 불가
4. 공공데이터 수집 파이프라인(`scraper.js` → odcloud.kr → DB) 흐름이 없음

**보완 방법**
- ASCII 또는 간단한 화살표 다이어그램으로 서비스 연결도 1장 추가
- Object Storage는 "향후 확장 예정" 한 줄로 명시해 질문 대비
- `scraper.js:79-95`의 `mapApiRow` 출력 필드를 테이블로 정리 (= 사실상 DB 스키마)

2. 모바일(390px)과 데스크탑(1000px+) 레이아웃 차이에 대한 설명 없음

**빠진 것**
1. 테스트 7건이 실제로 PASS인지 확인되지 않음 — 실행 결과 로그 없음
2. 배포 후 HTTP 200 접속 확인 증거(curl 결과, 서버 로그)가 없음

## 보안팀장

**빠진 것 (위험도 순)**

1. **PPT_EVIDENCE.md 자체에 실서버 IP(`223.130.131.194`)가 노출됨**
   - 섹션 9에서 "발표 자료에서 제거"라고 쓰면서, 같은 파일 안에 IP를 적었음 — 자기모순
   - 이 파일이 git에 올라가면 IP 공개됨

2. **`server.txt`의 .gitignore 포함 여부 미확인**
   - git 추적 중이면 비밀번호가 원격 저장소에 이미 올라간 상태일 수 있음

3. **git 커밋 히스토리 민감정보 스캔 결과 없음**
   - 배포 스크립트 작성·삭제 과정에서 키값이 커밋에 포함됐을 가능성

**보완 방법**
- PPT_EVIDENCE.md 섹션 9에서 실제 IP 값 삭제, "운영 서버 IP"로 교체
- `git status server.txt` 또는 `.gitignore` 확인해서 추적 여부 점검
- `git log -p | grep -E "(API_KEY|PASSWORD|Bearer|nv-)"` 로 히스토리 스캔 1회 실행
