## 구현할 최소 기능 3개

Day7 디자인을 바탕으로 Day8에서 실제로 코드에 반영할 항목이다.  
3개를 넘기면 시간이 부족하다 — 이 3개만 완성한다.

### 기능 1 — BRIEF 입력 완료 시각화

| 항목 | 내용 |
|------|------|
| 파일 | `style.css`, `runbook.js` |
| 변경 내용 | 입력 완료된 질문 카드에 `.q-block--filled` class 추가 → 왼쪽 초록 테두리 |
| 완료 기준 | Q1에 값 입력 → 카드 배경색 변경, Q1 지우기 → 원래대로 복원 |

### 기능 2 — 복사 버튼 피드백 개선

| 항목 | 내용 |
|------|------|
| 파일 | `runbook.js`, `style.css` |
| 변경 내용 | 복사 성공 시 버튼 아래에 "✅ 복사됨" 메시지 2초 표시 후 제거 |
| 완료 기준 | TEAM 단계에서 복사 버튼 클릭 → 메시지 표시 → 2초 후 사라짐 |

### 기능 3 — 모바일 375px 복사 버튼 위치 수정

| 항목 | 내용 |
|------|------|
| 파일 | `style.css` |
| 변경 내용 | `@media (max-width: 640px)`에서 `.copy-btn`을 프롬프트 블록 아래로 이동 |
| 완료 기준 | DevTools 375px에서 버튼이 텍스트 위에 겹치지 않음 |

## OMC Team으로 다시 검토할 질문

아래 질문 중 하나를 골라 `/oh-my-claudecode:team 3:executor` 로 재검토한다.  
(발표 Q&A 대비용 — 미리 검토해두면 날카로운 질문에 답할 수 있다)

```
/oh-my-claudecode:team 3:executor "OMC + Superpowers 런북 앱 최종 검토.

질문 1 (기획자에게): BRIEF 단계 7개 질문 중 초보자가 답하기 가장 어려운 질문은
어느 것이고, 어떻게 바꾸면 더 쉬워질까?

질문 2 (구현자에게): execCommand('copy') deprecated 위험을 지금 당장 고쳐야 하나,
아니면 현 시점에서 허용 가능한 기술 부채로 남겨도 되나?

질문 3 (검토자에게): 이 앱을 처음 보는 학생이 가장 먼저 막히는 지점은 어디인가?
UX 개선 우선순위 1가지만 제안해줘.

결과물: 각 역할의 답변 요약 3줄 이내."
```

## Superpowers로 완료 검증할 항목

발표 직전 `Superpowers verification-before-completion`으로 최종 확인한다.

```
Superpowers verification-before-completion 기준으로 다음 항목을 확인해줘.

확인 항목:
1. BRIEF → TEAM 탭 이동 시 Q1 답변이 프롬프트에 실제로 반영되는가
   (확인 방법: Chrome 콘솔에서 state.answers.q1 출력 후 TEAM 프롬프트 텍스트와 비교)

2. localStorage 저장 후 새 탭에서 열어도 데이터가 유지되는가
   (확인 방법: 새 탭 → index.html 열기 → BRIEF 확인)

3. 복사 버튼 피드백 메시지가 DOM에 추가됐다가 2초 후 제거되는가
   (확인 방법: DevTools Elements 탭에서 copy-ok span 추가/제거 관찰)

각 항목을 실제로 확인한 뒤, 확인됨 / 미확인 / 실패 로 기록해줘.
추측으로 '될 것 같다'고 쓰지 마.
```
