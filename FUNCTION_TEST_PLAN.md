## 테스트 환경

- Node.js 버전: v24.12.0 (요구사항 Node 18+, 외부 npm 패키지 없음)
- 테스트 실행 명령:
  - 구문 검사: `node --check scholarship-core.js && node --check runbook.js && node --check server.js`
  - 단위 테스트: `node --test tests/*.test.mjs`
  - 서버 실행: `node server.js` (기본 `http://127.0.0.1:8787`)
- 환경변수 (`.env`, 값은 절대 기록 금지):
  - `PORT` (기본 8787)
  - `CLOVA_STUDIO_API_KEY=<redacted>` — 비어 있으면 로컬 fallback 경로로 동작
  - `CLOVA_STUDIO_REQUEST_ID=<redacted>` — 선택값
- 기존 자동 테스트: 7건 통과 (`tests/scholarship-core.test.mjs` 5건, `tests/server.test.mjs` 2건)
- 데모 데이터: `data/scholarships.json` 8건 (verified 3건 / unverified 5건)

## 기능별 테스트 목록

### F1. 장학금 데이터 로드

정상 흐름: 페이지 접속 → `data/scholarships.json` fetch → 카드 8개 + 메트릭 표시 → 새로고침 시 동일 재현

| 케이스 | 입력 | 기대 결과 | 상태 |
|--------|------|-----------|------|
| 정상 로드 | `node server.js` 후 `/` 접속 | 카드 8개, "샘플 공고 8" 메트릭 | 수동 확인됨 (VERIFY.md) |
| 서버 미실행(파일 직접 열기) | `index.html` 직접 열기 | fetch 실패 → 빈 상태 안내 문구 표시, 콘솔 외 오류 없음 | 미작성 |
| JSON 손상 | 잘못된 JSON 응답 | "샘플 데이터를 불러오지 못했어" 빈 상태 문구 | 미작성 |

오류/빈 상태: 로드 실패 시 `empty-state` 문구만 있음 — 재시도 버튼 없음 (Phase 2 개선 항목).
발표 데모 경로: 데모 시작 전 `node server.js` 실행 → 첫 화면에서 메트릭 3종(샘플 공고/지원 가능/저장)이 0이 아닌지 확인.

### F2. 조건 필터 검색

정상 흐름: 필터 입력 → (input 이벤트마다 즉시 재계산, 별도 로딩 없음) → 결과 갱신 → "초기화" 클릭 시 전체 복원

| 케이스 | 입력 | 기대 결과 | 상태 |
|--------|------|-----------|------|
| 키워드 | `생활비` | 5개 결과 (VERIFY.md 기준) | 수동 확인됨 |
| 키워드+학교+유형 복합 | 키워드·학교·기관유형·구간·금액·상태 동시 | 조건 모두 만족하는 항목만 | 자동 테스트 있음 |
| all-level 매칭 | 학생 구분 `학부생`, all-level 데이터 | all 항목도 학부생 결과에 포함 | 자동 테스트 있음 |
| 학자금 지원구간 경계 | 구간 `6`, 데이터 max 6 | 포함됨 (경계 포함) | 미작성 |
| 학자금 지원구간 초과 | 구간 `7`, 데이터 max 6 | 제외됨 | 미작성 |
| 구간 빈값 | 구간 미입력 | 전체 통과 (필터 미적용) | 미작성 |
| 최소 금액 경계 | `3000000`, 데이터 300만 원 | 포함됨 (`<` 비교이므로 동일값 통과) | 미작성 |
| 공식 검증만 체크 | `verifiedOnly` on | verified 3건만 표시 | 미작성 |
| 빈 결과 | 키워드 `존재하지않는단어` | "조건에 맞는 샘플 장학금이 없어" 문구, 결과 0개 | 미작성 |
| 초기화 | 필터 적용 후 "초기화" | 전체 8개 복원, 입력값 모두 리셋 | 미작성 |
| 상태 정렬 | 상태 `전체` | open → upcoming → closed 순, 동순위는 금액 내림차순 | 미작성 |

오류/빈 상태: 빈 결과 문구 표시 확인됨(코드상). 숫자 필드에 음수/문자 입력 시 동작 미정의 — 확인 필요.
발표 데모 경로: 키워드 `생활비` 입력 → 결과 5개로 줄어드는 것 시연 → 초기화로 복원. 즉시 반응(input 이벤트)이므로 검색 버튼 누르기 전에 결과가 바뀌는 점을 멘트로 안내.

## 실패 케이스 목록 (우선 보완 순)

1. **`GET /.env` 노출 가능성** — `staticFilePath()`가 확장자 화이트리스트 없이 ROOT 전체를 서빙. `.env`에 `CLOVA_STUDIO_API_KEY=<redacted>`가 있는 환경이면 키 유출. **테스트로 즉시 확인 필요, 노출 시 Phase 2 최우선 수정.**
2. localStorage 손상 시 `runbook.js:7` `JSON.parse` 예외로 앱 전체 미기동 가능성.
3. CLOVA fetch 타임아웃 부재 — 무응답 시 연결 무한 대기 (Phase 2 수정 예정).
4. `/api/summary` 요청 수 제한 없음 — 키 설정 환경에서 크레딧 고갈 위험 (Phase 2 수정 예정).
5. 요약 버튼 연속 클릭 시 중복 요청 방지 없음.
6. 데이터 로드 실패 시 재시도 수단 없음 (새로고침만 가능).

- `/api/summary` HTTP 레벨 — 400 경로 3종 (빈 text, 35,000자 초과, 비JSON)
